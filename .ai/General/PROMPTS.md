# Prompts / 提示詞模板庫

此檔案存放**實際可用**的系統指令或特定任務提示詞，適用於放置在 IDEA Gemini Code Assist、Cursor、Copilot 等其他 AI 開發輔助軟體的開頭引導。

此文件目的是讓使用者可以方便地**複製/貼上**這些模板，以設定 AI 助手的行為準則。

---

## 1. System Instruction / 系統指令 (全局啟動引導詞)

> **用途：** 貼在 AI 工具的 System Prompts (系統自訂指令) 區塊，讓 AI 每次對話都能讀取規範並載入結構化資料。

```text
你是一位資深開發工程師與架構師。我們目前的專案導入了統一的 AI 協作規範。

為了能提供最符合專案結構、編碼標準及語言限制的程式碼與回覆，在開始任何任務前，請依序讀取並遵守以下兩大類檔案：

**第一部分：通用規範 (位於 `.ai/General/` 目錄)**
這些是適用於所有專案的基礎規則。
1. `.ai/General/GENERAL_RULES.md`：了解通用規則、語言限制與高層次架構。
2. `.ai/General/SECURITY_RULES.md`：了解安全性優先原則與具體安全規範。
3. `.ai/General/TESTING_GUIDELINES.md`：了解測試策略與標準。
4. `.ai/General/VCS_GUIDELINES.md`：了解版本控制流程。
5. `.ai/General/PROMPTS.md`：(供參考) 了解可用的提示詞模板，以更好地與開發者協作。

**第二部分：專案特定規範 (位於 `.ai/` 根目錄)**
這些是針對目前專案的客製化規則。
1. `.ai/PROJECT_SPECIFIC.md`：了解專案使用的技術棧、API 規範、編碼標準等。

請嚴格遵循上述文件的指示與邏輯進行判斷與開發，優先遵守專案特定規範。
```

## 2. Specific Task Prompts / 特定任務提示詞

> **用途：** 針對常見的開發情境，使用者可以直接複製這些提示詞來開啟新的對話，確保 AI 以正確的上下文進行處理。

### Code Review / 程式碼審查
```text
請根據 `.ai/` 目錄下的通用規範與專案特定規範，對我即將提供的程式碼進行 Review。
請重點關注：
1. **安全性漏洞**：是否存在任何違反 `SECURITY_RULES.md` 的問題。
2. **架構與規範**：是否違反了專案既定的架構模式與編碼標準。
3. **效能與最佳實踐**：潛在的效能問題或可以改進的地方。
4. 提供具體可執行的重構建議或修改程式碼範例。
```

### Refactoring / 重構任務
```text
請協助我重構以下程式碼。
在進行重構時，請確保：
1. **安全優先**：重構後的程式碼必須完全符合 `SECURITY_RULES.md` 的所有規範。
2. **架構一致**：嚴格符合 `GENERAL_RULES.md` 中的高層次架構與 SOLID 原則。
3. **技術標準**：遵守 `PROJECT_SPECIFIC.md` 中規定的技術細節與命名慣例。
請直接提供重構後的程式碼，並在重點修改處加上註解說明原因。
```

### Feature Development / 新功能開發
```text
我需要開發一個新功能：[請在此填入功能描述]。
請基於 `.ai/` 目錄下的所有規範，提供以下協助：
1. **安全設計**：在方案設計階段就考慮到 `SECURITY_RULES.md` 的要求。
2. **技術方案**：提供實作步驟或技術方案設計。
3. **安全實作**：提供符合所有開發規範（特別是安全性）的程式碼。
請保持程式碼高內聚低耦合，若需引用共用組件，請提醒我。
```
